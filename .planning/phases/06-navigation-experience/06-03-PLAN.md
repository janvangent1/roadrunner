    Create TileCacheManager.kt in package `com.roadrunner.app.data.tilecache`:
    - Annotate `@Singleton`, inject `@ApplicationContext context: Context`
    - `fun isTilesCached(routeId: String): Boolean`: check if directory `filesDir/osmdroid/tiles/Mapnik/10/{x}/{y}.png` has any files for the route. Simple heuristic: check `File(context.filesDir, "osmdroid/tiles/Mapnik/10").listFiles()?.isNotEmpty() == true`. This is a loose check — exact per-route tracking is a v2 concern.
    - `fun enqueueTileDownload(routeId: String, minLat: Double, maxLat: Double, minLon: Double, maxLon: Double)`:
      - Build `Data` object with keys: `"routeId"`, `"minLat"`, `"maxLat"`, `"minLon"`, `"maxLon"` (all doubles via `Data.Builder().putDouble(...)`)
      - Build `OneTimeWorkRequest.Builder(TileCacheWorker::class.java).setInputData(data).build()`
      - Call `WorkManager.getInstance(context).enqueueUniqueWork("tiles_$routeId", ExistingWorkPolicy.KEEP, request)`

    Create TileCacheWorker.kt in package `com.roadrunner.app.data.tilecache`:
    - Extend `CoroutineWorker(appContext, params)` — constructor injection (WorkManager creates workers, not Hilt)
    - `override suspend fun doWork(): Result`:
      - Extract input data: `inputData.getDouble("minLat", 0.0)` etc. for the four bounds
      - For each zoom level z in 10..15:
        - Compute tile X/Y ranges using standard slippy map formula:
          ```kotlin
          fun lon2tile(lon: Double, zoom: Int) = ((lon + 180) / 360 * (1 shl zoom)).toInt()
          fun lat2tile(lat: Double, zoom: Int): Int {
              val rad = Math.toRadians(lat)
              return ((1 - ln(tan(rad) + 1/cos(rad)) / Math.PI) / 2 * (1 shl zoom)).toInt()
          }
          ```
          Note: for lat, minLat gives maxY tile and maxLat gives minY tile (Y axis inverted in tile coords).
          `val xMin = lon2tile(minLon, z); val xMax = lon2tile(maxLon, z)`
          `val yMin = lat2tile(maxLat, z); val yMax = lat2tile(minLat, z)`
        - For each (x, y) in xMin..xMax × yMin..yMax:
          - File path: `File(applicationContext.filesDir, "osmdroid/tiles/Mapnik/$z/$x/$y.png")`
          - Skip if file already exists
          - Create parent dirs with `mkdirs()`
          - Download: `OkHttpClient().newGet("https://tile.openstreetmap.org/$z/$x/$y.png")` with header `User-Agent: RoadrunnerApp/1.0`
          - On successful response (200): write body bytes to file. On non-200 or exception: log and continue (don't fail the whole job)
      - Return `Result.success()` after all zoom levels complete
    - Use `kotlinx.coroutines.Dispatchers.IO` is the default for CoroutineWorker — no explicit dispatcher needed.
    - Import `kotlin.math.ln`, `kotlin.math.cos`, `kotlin.math.tan`, `kotlin.math.PI`

    Note: OSMDroid's default tile cache path when configured with `getCacheDir()` in Phase 3 is the cache dir. However, OSMDroid also checks `OfflineTileProvider` which reads from a `.zip` or `.sqlite`. The approach here writes PNG files to `filesDir/osmdroid/tiles/Mapnik/{z}/{x}/{y}.png`. To make OSMDroid pick these up automatically, the TileCacheWorker should write to `context.cacheDir` path instead: use `File(applicationContext.cacheDir, "osmdroid/tiles/Mapnik/$z/$x/$y.png")`. This matches where OSMDroid's `MapTileFileSystemProvider` looks when cache is configured to `getCacheDir()`. Update the TileCacheManager.isTilesCached check accordingly to use `cacheDir`.
