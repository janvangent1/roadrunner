<task type="auto">
  <name>Task 2: Create GpxCryptoManager with StreamingAead keyset seeding</name>
  <files>android/app/src/main/java/com/roadrunner/app/crypto/GpxCryptoManager.kt</files>
  <action>
Create the `crypto/` package directory and `GpxCryptoManager.kt` as a Hilt `@Singleton`.

The class must:
1. On `init()`, check if a keyset named `roadrunner_gpx_keyset` already exists in SharedPrefs `roadrunner_gpx_prefs`. If not, deserialize `BuildConfig.TINK_KEYSET_B64` using `BinaryKeysetReader` (NOT `JsonKeysetReader` — the server uses binary-format keyset, not JSON). Wrap it with Android Keystore master key `android-keystore://roadrunner_gpx_master` and persist to SharedPrefs via `AndroidKeysetManager`.
2. Always obtain the `StreamingAead` primitive from the stored keyset handle.
3. Expose `fun decryptToByteArray(routeId: String, encFile: File): ByteArray` that opens a `DecryptingStream` with AAD = `routeId.toByteArray()` and calls `.readBytes()`. Returns the plaintext bytes. The `routeId` as AAD MUST match what the server used when encrypting (Phase 1 decision: `Buffer.from(routeId)` as AAD).
4. Expose `fun encryptedFileExists(routeId: String, filesDir: File): Boolean` — returns whether `File(filesDir, "gpx/$routeId.enc").exists()`.

Key implementation details:
- Use `StreamingAeadConfig.register()` (NOT `AeadConfig.register()`) before accessing the keyset
- Key template: `KeyTemplates.get("AES256_GCM_HKDF_4KB")`
- The init logic must be guarded — `AndroidKeysetManager.Builder()` with `.withSharedPref()` automatically persists after first creation, so calling `.build()` is idempotent after the first call
- For the keyset seeding from BuildConfig: use `Base64.decode(BuildConfig.TINK_KEYSET_B64, Base64.DEFAULT)` → `BinaryKeysetReader.withBytes(bytes)` → `CleartextKeysetHandle.read(reader)` to get the handle, then store it using `AndroidKeysetManager.Builder().withSharedPref(...).withMasterKeyUri(...).build()` to wrap it with Keystore
- Inject `@ApplicationContext context: Context` via Hilt

Imports needed:
```kotlin
import com.google.crypto.tink.BinaryKeysetReader
import com.google.crypto.tink.CleartextKeysetHandle
import com.google.crypto.tink.KeyTemplates
import com.google.crypto.tink.StreamingAead
import com.google.crypto.tink.integration.android.AndroidKeysetManager
import com.google.crypto.tink.streamingaead.StreamingAeadConfig
```

The `decryptToByteArray` function must close the decrypting stream properly (use `.use { it.readBytes() }`). Any exception from Tink (wrong key, corrupted file, wrong AAD) should propagate as-is — callers handle errors.
