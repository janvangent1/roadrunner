**RouteRepository.kt** — four changes:

1. Update the constructor to also inject `GpxCryptoManager` and `@ApplicationContext context: Context`:
   ```kotlin
   class RouteRepository @Inject constructor(
       private val apiService: ApiService,
       private val gpxCryptoManager: GpxCryptoManager,
       @ApplicationContext private val context: Context,
   )
   ```

2. Add `downloadAndStoreGpx(routeId: String): Result<Unit>`:
   - Create `File(context.filesDir, "gpx")` directory with `.mkdirs()` if it doesn't exist
   - Call `apiService.getRouteGpx(routeId)` — if not successful, return `Result.failure`
   - Pipe `response.body()!!.byteStream()` directly to `FileOutputStream(File(context.filesDir, "gpx/$routeId.enc"))` using `.use { input -> FileOutputStream(...).use { output -> input.copyTo(output) } }`
   - The bytes written are the raw encrypted blob from the server — NOT decrypted at any point
   - Return `Result.success(Unit)` on completion
   - Wrap in try/catch returning `Result.failure` on any exception

3. Add `getDecryptedGpx(routeId: String): Result<ByteArray>`:
   - Check if `.enc` file exists via `gpxCryptoManager.encryptedFileExists(routeId, context.filesDir)`. If not, call `downloadAndStoreGpx(routeId)` first. If download fails, return that failure.
   - Call `gpxCryptoManager.decryptToByteArray(routeId, File(context.filesDir, "gpx/$routeId.enc"))`
   - Return `Result.success(bytes)`
   - Wrap in try/catch returning `Result.failure` on any exception

4. No changes to existing methods.

CRITICAL: At no point should plaintext GPX bytes be written to a file. The `downloadAndStoreGpx` copies the raw (already-encrypted) server response bytes to disk. The `getDecryptedGpx` decrypts only in memory.
