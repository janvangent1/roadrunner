**POST /** (resolves to `/api/v1/admin/licenses`) — Grant a license.

Request body (Zod schema):
```typescript
{
  email: string,           // Lookup user by email
  routeId: string,         // UUID
  type: 'DAY_PASS' | 'MULTI_DAY' | 'PERMANENT',
  expiresAt: string | null // ISO datetime string; required for DAY_PASS and MULTI_DAY; null for PERMANENT
}
```

Logic:
1. Find user by email — return 404 `{ error: 'User not found' }` if no match
2. Verify the route exists — return 404 `{ error: 'Route not found' }` if not
3. Create `prisma.license.create(...)` with `revokedAt: null`, `linkedPurchaseToken: null` (reserved for v2)
4. Return 201 with the created license object (include user email and route title for readability)
5. Invalidate Redis cache key `license:{userId}:{routeId}` (redis.del)

**PATCH /:id** — Modify/revoke a license.

Request body (all fields optional, at least one required):
```typescript
{
  type?: 'DAY_PASS' | 'MULTI_DAY' | 'PERMANENT',
  expiresAt?: string | null,
  revoked?: boolean   // If true, sets revokedAt = new Date() (server clock); if false, clears revokedAt
}
```

Logic:
1. Find license by id — return 404 if not found
2. Build update data from provided fields
3. `prisma.license.update(...)`
4. Invalidate Redis cache key `license:{license.userId}:{license.routeId}` (critical: revocation must propagate within 60s per cache TTL)
5. Return 200 with updated license

**GET /** — List all licenses with optional filters.

Query params: `routeId?: string`, `userId?: string`. Return array of licenses with joined user.email and route.title.

```typescript
  include: {
    user: { select: { id: true, email: true } },
    route: { select: { id: true, title: true } },
  },
  orderBy: { createdAt: 'desc' },
})
```

**POST /check** (resolves to `/api/v1/licenses/check`) — Validate license before navigation.

Request body (Zod):
```typescript
{
  routeId: string,   // UUID of the route to navigate
  deviceId: string   // For future Play Integrity binding (v2); store but don't validate yet
}
```

Logic:

1. **Redis cache check** — check key `license:{request.user.sub}:{routeId}`.
   - Cache value is a JSON string `{ valid: boolean, expiresAt: string|null }`.
   - If cache hit and `valid === true`, skip DB query and proceed to step 3.
   - If cache hit and `valid === false`, return 403 immediately.
   - If cache miss, proceed to step 2.

2. **DB validation** — query:
   ```typescript
   const license = await prisma.license.findFirst({
     where: {
       userId: request.user.sub,
       routeId: params.routeId,
       revokedAt: null,
       OR: [
         { expiresAt: null },                        // PERMANENT
         { expiresAt: { gt: new Date() } },          // Not yet expired (server clock)
       ],
     },
   });
   ```
   - CRITICAL: `new Date()` uses the server clock — NEVER trust any client-provided timestamp.
   - If no license found, cache `{ valid: false }` with TTL 60s, return 403 `{ error: 'No valid license', code: 'LICENSE_INVALID' }`.
   - If license found, cache `{ valid: true, expiresAt: license.expiresAt }` with TTL 60s.

3. **Issue navigation session JWT** — sign a short-lived JWT (1-hour window):
   ```typescript
   const sessionExpiresAt = new Date(Date.now() + 60 * 60 * 1000); // 1 hour from NOW (server clock)
   const sessionToken = fastify.jwt.sign(
     {
       sub: request.user.sub,
       routeId,
       sessionType: 'navigation',
       issuedAt: new Date().toISOString(),  // Server clock
       sessionExpiresAt: sessionExpiresAt.toISOString(),
     },
     { expiresIn: '1h' }
   );
   ```

4. **Return:**
   ```json
   {
     "valid": true,
     "sessionToken": "<JWT>",
     "sessionExpiresAt": "<ISO datetime — 1 hour from now>",
     "licenseType": "PERMANENT|DAY_PASS|MULTI_DAY",
     "licenseExpiresAt": "<ISO datetime or null>"
   }
   ```
