  <action>
Create `backend/src/lib/tink.ts` — a module that wraps Tink's AES-256-GCM streaming encryption for GPX files.

**Package to use:** `tink-crypto` (the npm package — pure TypeScript port of Google Tink for Node.js/browser). Install: `npm install tink-crypto`.

**What to implement:**

```typescript
import { aead } from 'tink-crypto';

let keysetHandle: Awaited<ReturnType<typeof aead.keysetHandle>> | null = null;

/**
 * Initialize Tink with the keyset from the TINK_KEYSET_JSON env var.
 * TINK_KEYSET_JSON is a base64-encoded Tink JSON keyset (not raw JSON).
 * Call once at app startup.
 */
export async function initTink(): Promise<void> {
  const keysetJson = Buffer.from(process.env.TINK_KEYSET_JSON!, 'base64').toString('utf8');
  keysetHandle = await aead.keysetHandle(JSON.parse(keysetJson));
}

/**
 * Encrypt a GPX buffer in memory. Returns encrypted bytes.
 * NEVER writes plaintext to disk. The input Buffer is encrypted
 * entirely in memory and the encrypted Uint8Array is returned.
 *
 * @param plaintext - Raw GPX file bytes
 * @param associatedData - Optional AAD (e.g. route ID as UTF-8 bytes)
 */
export async function encryptGpx(plaintext: Buffer, associatedData?: Buffer): Promise<Buffer> {
  if (!keysetHandle) throw new Error('Tink not initialized — call initTink() first');
  const aeadPrimitive = await aead.primitive(keysetHandle);
  const encrypted = await aeadPrimitive.encrypt(
    new Uint8Array(plaintext),
    associatedData ? new Uint8Array(associatedData) : new Uint8Array(0)
  );
  return Buffer.from(encrypted);
}

/**
 * Decrypt an encrypted GPX buffer in memory. Returns plaintext bytes.
 */
export async function decryptGpx(ciphertext: Buffer, associatedData?: Buffer): Promise<Buffer> {
  if (!keysetHandle) throw new Error('Tink not initialized — call initTink() first');
  const aeadPrimitive = await aead.primitive(keysetHandle);
  const decrypted = await aeadPrimitive.decrypt(
    new Uint8Array(ciphertext),
    associatedData ? new Uint8Array(associatedData) : new Uint8Array(0)
  );
  return Buffer.from(decrypted);
}
```

Call `initTink()` in `backend/src/server.ts` before `server.listen()` — add the call there.

If `tink-crypto` API differs from the above (check the actual package docs), adapt accordingly — the KEY INVARIANTS that must be maintained are:
1. Encryption uses AES-256-GCM from the env-var keyset
2. No plaintext is written to disk at any point
3. The function is async and returns a Buffer

**backend/src/routes/routes.ts** — export `routeHandlers` as a Fastify plugin (use fastify-plugin). Register with `preHandler: [requireAuth]` at plugin level (all routes require auth).

Three handlers:

- `GET /` (resolves to `/api/v1/routes`) — `prisma.route.findMany({ where: { published: true }, select: { id, title, description, difficulty, terrainType, region, estimatedDurationMinutes, distanceKm, published, createdAt, updatedAt } })`. EXCLUDE `gpxEncrypted`. Also check Redis cache key `catalog` (TTL 5min) — serve from cache if fresh, otherwise query DB and set cache. Return array. Also include per-route catalog cache key in Redis: `catalog` → JSON string → TTL 300s.

- `GET /:id` — `prisma.route.findUnique({ where: { id }, include: { waypoints: { orderBy: { sortOrder: 'asc' } } } })`. EXCLUDE `gpxEncrypted` from the select (never send GPX in catalog/detail responses). Return 404 if not found.

- `GET /:id/gpx` — License check: query `prisma.license.findFirst({ where: { userId: request.user.sub, routeId: params.id, revokedAt: null, AND: [{ OR: [{ expiresAt: null }, { expiresAt: { gt: new Date() } }] }] } })`. If no valid license, return 403 `{ error: 'No valid license for this route' }`. If valid, fetch `prisma.route.findUnique({ where: { id }, select: { gpxEncrypted: true } })`. Return 404 if route not found or GPX not yet uploaded. Stream the encrypted bytes as response with Content-Type `application/octet-stream` and header `X-Encrypted: tink-aes256gcm`.

**backend/src/routes/adminRoutes.ts** — export `adminRouteHandlers` as a Fastify plugin. All routes use `preHandler: [requireAuth, requireAdmin]`.

Three handlers:

- `POST /` (resolves to `/api/v1/admin/routes`) — multipart form. Use `request.file()` from `@fastify/multipart` to read the GPX file field (`name="gpx"`). Read all bytes into a Buffer (do NOT save to disk — use `await pump(file.file, ...)` collecting into a Buffer). Encrypt with `encryptGpx(gpxBuffer, Buffer.from(routeId))`. Parse other fields from the multipart form: `title`, `description`, `difficulty` (validated against Difficulty enum), `terrainType`, `region`, `estimatedDurationMinutes` (int), `distanceKm` (decimal). If `waypoints` field is present (JSON string), parse and create waypoints in the same transaction. Create the route with `published: false` by default. Return 201 with the new route object (excluding gpxEncrypted). Invalidate the `catalog` Redis cache key.

- `PATCH /:id` — accepts JSON body with any subset of: `title`, `description`, `difficulty`, `terrainType`, `region`, `estimatedDurationMinutes`, `distanceKm`, `published`. Optionally re-accepts a GPX file via multipart (if `Content-Type: multipart/form-data` — re-encrypt and update). Return updated route (excluding gpxEncrypted). Invalidate `catalog` Redis cache key.

- `DELETE /:id` — delete route (waypoints cascade per schema). Return 204. Invalidate `catalog` Redis cache.
